# Product

## Register

product

## Users

Internal team at Mark III Brokerage — brokers and staff learning BenSelect JScript.NET scripting as part of their job. They are not software developers; they are insurance/benefits professionals who need to write and understand scripts to configure BenSelect products. They use this LMS on a desktop browser during work hours, in a task-completion mindset: get through the module, pass the quiz, move on.

## Product Purpose

A structured, self-paced training platform that teaches BenSelect JScript.NET scripting from first principles through 14 modules. Success = a learner who can write real OnEligible / OnCalc / OnSave scripts without assistance. The LMS tracks progress, enforces module sequencing, and provides a quiz gate at each module to confirm understanding before unlocking the next.

## Brand Personality

Professional, technical, clear. The interface should feel like a well-made internal tool built by someone who cares — not a SaaS product trying to impress, not a generic HR LMS, not a consumer ed-tech app. Trustworthy and competent.

## Anti-references

- Generic corporate LMS (Cornerstone, TalentLMS): avoid gray, dated, checkbox-heavy layouts with low visual density and zero design intent.
- Consumer ed-tech (Duolingo, Coursera): avoid badges, gamification, overly playful UI — this is professional technical training for working adults.
- Heavy analytics dashboards: avoid chart-first layouts that compete with the learning content.

## Design Principles

1. **Tool disappears, content leads.** The UI chrome should never compete with code examples, explanations, or exercises.
2. **Progress is the reward.** Track completion clearly and honestly — no fake badges. A clean progress indicator and locked/unlocked state is enough.
3. **Code is first-class.** Code blocks, the script editor, the glossary — these are the core artifacts. They should look precise and readable, not decorative.
4. **Clarity over personality.** Every label, heading, and instruction should be the clearest possible version of itself.
5. **Respect the learner's time.** Minimize friction for the primary path: open module → read → practice → quiz → continue.

## Accessibility & Inclusion

WCAG AA minimum. Desktop-primary, office environment. Keyboard navigation required (script editor, navigation). Reduced motion support required.
